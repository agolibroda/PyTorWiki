
Группы

Группа  - это возможность структурирования материалов сайта по "интересам", "направлениям"....

Группа объединяет участников в работе над статьями некой тематики.
Статьи в группе могут быть "публичными" 'pbl' - доступными для прочтения не участниками группы, 
и "внутренними" "grp" - доступными к прочтению исключительно Участниками.

Любой Участник группы может положить в группу и "персональные" ('sol') статьи, 
- таким образом можно структурировать собственные статьи по различным тематикам. 

Группа может быть 
    - публичная - 'pbl' - любой посетитель может открыть "рабочий стол групы", 
    он увидит там библиотеку (список статей) и список участников группы,  
     Сможет читать публичные материалы группы; 
    - закрытая - 'shut' - Вход на рабочий стол группы возможен только участникам группы, 
    (после авторизации, и из персонального рабочего стола) в такой группе не может быть "pbl" статей; 


Группа представляет сбой:
    - "заголовок" - название, описание, дата создания, Автор - создатель группы,
    - список участников
    - библиотеку статей; у каждой группы одна библиотека.
    
Операции над  Группой:
	- Создание группы;
	- Просмотр списка всех групп;
    - Просмотр одной группы:  
    	- Описание
		- Список участников группы
		- Список статей (библиотека)

Операции внутри Группы:
        
    - Добавлять (удалять) участников в группу может только Администратор Группы
    - Добавлять (удалять) статьи в библиотеку может любой Участник Группы
        - статья находится в библиотеке и имеет флаг: 
            - "pbl" - для всеобщего доступа;  
            - "grp" - только для группы, такие статьи будут ЗАКРЫТЫМИ!!!! для "читателей", не принадлежащих группе


Процедура создания новой группы:
 	При создании новой группы, Создатель группы становится ее Администратором.
	Запись о создании группы размещается в таблицах "dt_headers" и "groups" 
	Запись о вступлении в группу Администратора добавляется в таблицу "members";
              
Процедура работы с Ключами:
	Создается уникальная пара RSA-ключей, 
	Публичный ключ помещается в заголовок группы,  
	персональный  - размещается в списке "members", 
	Приватный ключ группы закрывается Публичным ключем Создателя группы, 
	и добавляется в соответствующее поле таблицы "members";

При создании новой статьи с флагом "grp" текст статьи закрывается "публичным ключем Группы".

При добавлении нового участника в групу создается новая копия приватного ключа группы, 
которая закрывается	публичным ключем нового участника, и добавлется к записи этого участника 
в таблице "members"

Когда Участник Группы открывает страницу группы (переходит на рабочий стол группы) 
в профиль "Авторизованного Автора" добавляется значение его копия приватного ключа группы;
После этого пользователь сможет читать и редактировать все статьи из групповой библиотеки, 
имеющие флаг "grp"
